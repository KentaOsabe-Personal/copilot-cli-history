# プロダクト概要

updated_at: 2026-04-27

このリポジトリは、**GitHub Copilot CLI のローカル会話履歴を参照・検索できるアプリ**を育てるための土台です。  
ユーザーが日常的に使う Copilot CLI のセッション履歴を、CLI の再開操作に依存せず、アプリから一覧・詳細・検索できる体験を目指します。

## コア機能

- **ローカル履歴の読取**: `COPILOT_HOME` または `~/.copilot` 配下のセッションデータを起点に扱う
- **複数保存形式の吸収**: 現行の `session-state` と旧形式 `history-session-state` を同じプロダクト文脈で扱う
- **読取専用 API 契約**: セッション一覧・詳細を read-only の JSON API で返し、UI から再利用しやすい共通 schema を保つ
- **セッション可視化**: セッション一覧、会話タイムライン、ツール呼び出しや変更情報の参照を支える
- **検索と絞り込み**: リポジトリ、ブランチ、日付、モデルなどの軸で履歴を再利用しやすくする
- **正規化ストア活用**: raw files を一次ソースにしつつ、アプリ側 DB を検索・再利用のための補助層として使う

## 主なユースケース

- 過去セッションを**あとから読み返して**、プロンプト・応答・ツール実行の流れを確認する
- リポジトリやブランチをまたいで、**どの会話で何をしたか**を探す
- 実装の経緯や意思決定を履歴からたどり、**開発コンテキストを復元する**
- 将来の UI/API 実装で、履歴データを**検索・比較・再表示しやすい形**に整える

## 価値提案

- **ローカルファースト**: 会話履歴の一次ソースはローカルの session files であり、外部サービス前提にしない
- **CLI 非依存の参照体験**: `copilot --resume` のような順次操作とは別に、閲覧と探索に特化した UI を提供できる
- **形式変更への耐性**: 不明イベントや将来の schema 差分を吸収できるよう、raw data を尊重する
- **開発しやすい段階構成**: まず Docker ベースの開発基盤を整え、その上に reader / API / UI を積み上げる

## プロダクト原則

### 1. Raw files を正本とする

Copilot CLI の保存ファイルが一次ソースです。  
アプリ側の DB や検索インデックスは再生成可能な補助層として扱います。

### 2. まずはローカル閲覧を優先する

初期スコープは、手元の会話履歴を安全に読み返せることです。  
外部共有や送信を前提に設計せず、ローカル用途を基準に判断します。

### 3. 新旧形式の共存を前提にする

現行形式だけに寄せず、履歴保存形式の移行期間を吸収できる reader を前提にします。  
フォーマット差分は UI ではなく取り込み層で吸収します。

### 4. 仕様駆動で段階的に拡張する

このリポジトリでは `.kiro/specs/` を使って要件・設計・タスクを管理します。  
新機能は、履歴 reader、API、UI を一気に広げるより、仕様ごとに小さく積み上げます。

### 5. 壊れたデータも隠さず扱う

履歴ファイルの欠損や部分的な破損があっても、読めた部分までを返せる設計を優先します。  
完全失敗とセッション単位の degraded 状態を分け、利用者が原因を追えるよう issue 情報を残します。
